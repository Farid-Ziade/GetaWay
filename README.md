# GetaWay v2

A production-minded Flutter app for planning smart getaways: secure sign-in, maps and nearby places, weather-aware suggestions, AI-generated weekend plans (via your own backend only), budget control, and saved trips without repeating the same recommendations.

## Current status (Phase 1)

Phase 1 focuses on **project layout**, **central theme**, **named routing**, a **splash → login / home shell**, **documentation**, and **git hygiene** for secrets.

| Area | Status |
|------|--------|
| Feature-based `lib/` layout | Done |
| App theme (Material 3 + Poppins) | Done |
| Named routes (`/login`, `/home`) | Done |
| Splash + placeholder home | Done |
| Login UI (validation, phone field) | Present; Firebase wiring is Phase 3 |
| `.env` ignored; `.env.example` in repo | Done |
| Maps / weather / AI / backend | Not started |

## Tech stack (so far)

- [Flutter](https://docs.flutter.dev/) (Dart SDK ^3.11)
- [Firebase Core](https://firebase.google.com/docs/flutter/setup) + [Firebase Auth](https://firebase.google.com/docs/auth/flutter/start) (dependencies added; full auth flow in Phase 3)
- [Google Fonts](https://pub.dev/packages/google_fonts)
- [Google Sign-In](https://pub.dev/packages/google_sign_in) (dependency present; UI hookup in Phase 3)
- [intl_phone_field](https://pub.dev/packages/intl_phone_field) for phone input

## Project structure

```text
lib/
├── main.dart                 # Firebase init, runApp
├── firebase_options.dart     # Generated by FlutterFire (Android configured)
├── app/
│   └── getaway_app.dart      # MaterialApp, routes, theme
├── core/
│   └── theme/
│       └── app_theme.dart    # Shared ThemeData
├── features/
│   ├── splash/…              # Splash + auth gate
│   ├── auth/…                # Login screen + AuthService
│   └── home/…                # Signed-in placeholder home
└── shared/                   # Reserved for reusable widgets/utils (empty for now)
```

## Run the app

1. Install [Flutter](https://docs.flutter.dev/get-started/install) and accept Android licenses if you target Android.
2. From the project root:

   ```bash
   flutter pub get
   flutter run
   ```

3. Optional: copy `.env.example` to `.env` when you add environment-driven config (no secrets in git).

## Product name vs package name

The app is branded **GetaWay**; the Dart package name in `pubspec.yaml` remains `getaway_app` (changing it affects imports and tooling). You can rename later with a dedicated migration step.
